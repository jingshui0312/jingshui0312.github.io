💡 核心理念

人类掌舵，AI 执行

OpenAI 团队进行了一项激进实验：用 0 行人工代码构建完整的软件产品。 所有代码（应用逻辑、测试、CI配置、文档、可观测性）都由 Codex 生成。 结果显示效率提升 10 倍，5 个月生成百万行代码，服务数百名用户。

🎯 核心洞察

📍 Context Management

给 AI 地图而非说明书。100 行 AGENTS.md + 结构化 docs/，而非 1000 页手册。

🏗️ Architecture First

严格分层架构：Types → Config → Repo → Service → Runtime → UI。机械执行约束。

⚡ Golden Principles

共享工具包优于手写助手。不要 YOLO 式探测数据。背景任务持续扫描偏差。

🤖 Agent Autonomy

AI 可端到端驱动功能：11 个步骤从验证到部署，无需人工干预。

🗑️ Garbage Collection

技术债务像高利贷。持续小增量偿还优于周期性大重构。每日审查，自动合并。

👥 Human Layer Shift

人类工作在更高抽象层：优先级、验收标准、验证结果。Agent 写代码。

📅 时间线

2025年8月

首次提交到空仓库。Codex CLI + GPT-5 生成初始脚手架（仓库结构、CI配置、格式规则）。

早期阶段

进度比预期慢。原因：环境未充分指定。Agent 缺乏工具、抽象和内部结构。

5个月后

百万行代码、1500+ PR、数百用户。团队从 3 人增长到 7 人，吞吐量反而提升。

当前状态

Codex 可以端到端驱动新功能。单个 prompt 触发完整开发流程（验证→修复→测试→部署）。

🌐 影响领域

💻 Software Development

重新定义软件工程师角色。从"写代码"到"设计环境"。工作在更高抽象层。

🤖 AI Engineering

Agent-first 开发模式。Repository-optimized for AI legibility。Context management 成为关键技能。

🚀 Engineering Productivity

10 倍效率提升。3.5 PR/人/天。修正成本低，等待成本高。最小化阻塞。

⚙️ Automation

端到端自主功能开发。Agent-to-agent review。6+ 小时无人值守运行。

💡 核心洞察

"Building software still demands discipline, but the discipline shows up more in the scaffolding rather than the code."

纪律仍然重要，但体现在脚手架（工具、抽象、反馈循环）而非代码本身。
